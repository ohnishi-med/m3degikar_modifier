このプロジェクトでは、
クラウド型カルテのｍ３デジカルの利便性を向上させるJS群である。

現在4つのJSがある。
１．セル塗り分け
受付状況に応じてセルのカラーリングを変更する

２．検査オーダー記録
オーダー発行のボタンを押したことを視認できるようにする

３．日数計算
次回の予約日・日数の計算およびインスリンの容量計算を行う

４．採用薬剤マスタ
院内採用薬のマスタを作成し、表示形式を調整することで、
採用薬と非採用薬を分かりやすく区別し、診療を迅速に行うことです。


TampermonkeyでJSを動作させている。
仕様はJSファイルを参照すること。

GoogleSpreadSheetで医薬品マスタを保持
院内採用のFlagをA列におき、１は採用
シート２に採用薬のみのマスタを保持。（=QUERY(MedicationMasterR8_4!A:F, "SELECT F WHERE A = 1", 1)）

スプレッドシートのGASはGas.txtを参照すること

現在の課題と対応方針
・採用登録を行った際、スプレッドシートの関数(QUERY)再計算に時間がかかるため、画面への反映にタイムラグが生じる。
  - 対応方針: JS側で「オプティミスティック（楽観的）UI更新」を導入する。GASへの登録リクエストが成功した時点（あるいは送信時点）で、ローカルの採用薬配列に対象薬を追加し画面を再描画することで、体感的なタイムラグを解消する。

## 自動更新・配信方針 (10台以上のPCへの対応)
院内の10台以上あるPCへ修正を自動で行き渡らせるため、各スクリプトは GitHub の Raw URL を参照した「自動更新」に対応させています。
- スクリプトに変更を加えた際は、ヘッダーの `@version` をカウントアップして GitHub へプッシュしてください。
- すると、各PCのTampermonkeyが定期巡回時に最新版を自動的にダウンロードします。
