# 要件定義書: M3デジカル臨床データ連携ツール群

## 1. プロジェクト概要
M3デジカル電子カルテにおいて、外部データ（Googleスプレッドシート）や検査結果からの自動計算結果を、カルテの入力フィールドにシームレスに流し込むためのTampermonkeyスクリプト群。

## 2. 長谷川式 (HDS-R) データ連携機能
### 臨床要件
- **目的**: 外部で実施された長谷川式認知症スケール (HDS-R) の結果をカルテに転記する。
- **データ項目**:
    - 点数 (0-30点)
    - 検査年月日 (YYYY/MM/DD)
- **データの出所**: 指定のGoogleスプレッドシート
    - A列: 検査年月日
    - B列: カルテID (検索キー)
    - N列: 点数

### UI/UX要件
- **起動ボタン**: カルテヘッダー（ツールバー）に設置。緑色のデータベースアイコン。
- **操作フロー**:
    1. カルテ画面右上のボタンをクリック。
    2. カルテIDを自動抽出（`.css-ustlin` 内の `span` から取得）。
    3. GAS API経由でスプレッドシートを検索。
    4. 結果をプレミアム感のあるモーダルダイアログ（Glassmorphism）で表示。
    5. 「登録」ボタン押下で、カルテ内の「HDS-R」または「長谷川」ラベルのフィールド、および「検査年月日」フィールドに値を自動流し込み。

### 技術仕様
- **フロントエンド**: JavaScript (Tampermonkey)
- **バックエンド**: Google Apps Script (Web App)
- **データ取得**: `GM_xmlhttpRequest` によるクロスドメイン通信。

## 3. 推定塩分摂取量・FE指標計算機能
### 臨床要件
- **目的**: 尿中・血清検査結果から各種臨床指標を算出する。
- **計算項目**:
    - 推定塩分摂取量 (田中式、川崎式)
    - FENa (ナトリウム排泄率)
    - FEUn (尿素窒素排泄率)
    - FECa (カルシウム排泄率)
- **医学的根拠**: 各指標の標準的な計算式に基づく（年齢、性別、身長、体重、sCr, uCr, sNa, uNa等を使用）。

### UI/UX要件
- **起動ボタン**: ツールバーに設置。緑色の塩アイコン。
- **操作フロー**:
    1. 「検査結果」タブを自動クリックして最新データを取得。
    2. SOAP等から身長・体重を抽出。
    3. 計算結果をダイアログで表示。
    4. 欠損データがある場合は、不足項目をエラー表示。
    5. 登録実行で、各臨床指標の入力フィールドに値を自動入力し、保存。

## 4. 共通仕様
- **React対応**: 入力値の反映には React の `setNativeValue` (イベント発火) を使用。
- **開発環境**: `dev-proxy.user.js` を介してローカルファイルを読み込むことで、デバッグ効率を最大化。
- **ファイル整理**:
    - `js/`: 各種 Tampermonkey スクリプト本体
    - `gas/`: スプレッドシート連携用 GAS ソース
