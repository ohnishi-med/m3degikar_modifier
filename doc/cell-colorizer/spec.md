# 受付セル塗り分け (cell-colorizer) 仕様書

## 1. 概要
M3デジカルの受付一覧（Reception）画面において、患者のステータスや属性（年齢、保険、メモ等の内容）に応じて行全体の背景色や特定セルの色を自動で変更・警告する Tampermonkey スクリプトです。

## 2. 目的
受付スタッフおよび医師が、患者の属性（小児科対応の必要性、公費・保険の入力漏れ、発熱外来などの特記事項）を一目で視認できるようにし、カルテ業務の効率化と入力ミスの防止を図ります。

## 3. 主要機能と判定ロジック

### 3.1. 医師名セル（固定色・巡回色）
*   **対象カラム**: 医師
*   **仕様**: 
    *   **固定色**: 特定の医師（大西: アリスブルー、坂口: ラベンダーブラッシュ、久保: アイボリーグリーン）には常に決まった色が割り当てられます。
    *   **巡回色**: 上記以外の医師には、事前に定義されたパレット色（シーシェル、ミントクリームなど）が順番に割り当てられます。
*   **優先順位**: 最高（他の背景色を上書きします）。

### 3.2. 行全体のベース色付け（メモベース）
*   **対象カラム**: 受付メモ
*   **仕様**:
    *   `受付メモ` に **「オンライン」** が含まれる場合：行全体を淡いパープル（Lavender）に設定。
    *   `受付メモ` に **「発熱外来」** が含まれる場合：行全体を淡いピンク/赤（MistyRose）に設定（※オンラインより優先）。

### 3.3. 警告ルールの適用（セルの色付け）

#### Rule A: 診療科と年齢の不一致（小児科ルール警告）
*   **対象カラム**: 診療科、年齢
*   **警告色**: 薄いオレンジ
*   **判定条件**:
    1.  **15歳未満**の患者が、**「小児科」「皮膚科」「自由診療」以外**の科を受診している場合。
    2.  **15歳以上**の患者が、**「小児科」**を受診している場合。
*   **適用箇所**: 該当する場合、「診療科」セル（および「年齢」セル）を警告色に変更します。

#### Rule B: 公費入力漏れ警告（子供医療費）
*   **対象カラム**: 保険
*   **警告色**: 薄い黄色
*   **判定条件**:
    以下の **すべて** を満たす場合、公費入力漏れとみなして警告。
    1.  **18歳年度末（3月31日）まで** の患者であること。（生年月日または年齢から算出）
    2.  診療科が **「自由診療」以外** であること。
    3.  保険テキストに **公費キーワード（こども, 子, ひとり, 生活保護, 親, 障害）が含まれていない** こと。
    4.  受付メモ、および **患者メモ** のいずれにも **「県外」という文字が含まれていない** こと。（県外在住の場合は子供医療費が使えないため警告対象外）

#### Rule C: 保険入力不備の検知
*   **対象カラム**: 保険
*   **警告色**: 薄い黄色
*   **判定条件**:
    1.  保険が「保険無し」だが、診療科が「自由診療」ではない場合（保険入力漏れ）。
    2.  保険が「保険無し」ではないが、診療科が「自由診療」の場合（自費と保険の矛盾）。

## 4. 技術的特記事項
*   **カラムの自動検出**: テーブルヘッダー（`thead th`）を走査し、「ステータス」「生年月日」「年齢」「保険」「診療科」「医師」「患者メモ」「受付メモ」のインデックスを動的に取得します。表示順序が変更されても追従可能です。
*   **React (SPA) 対応**: `history.pushState/replaceState` をフックし、ソフトナビゲーションによる画面遷移時にも処理が再起動する仕組みになっています。
*   **DOM監視 (MutationObserver)**: テーブルの変更（行の追加、入力要素の変更等）を監視し、リアルタイムに色付けルールを再適用します。
*   **生年月日チェック**: 18歳判定をより厳密に行うため、現在の日付と生年月日から「高校卒業年度の3/31」を計算します。「生年月日」カラムが非表示の場合は年齢ベースの簡易判定にフォールバックし、トースト通知で警告を出します。
